# Battle UI Grid Spec v0.1

本文档定义当前训练场和战斗 UI 的第一版网格规划。目标不是一次性锁死最终美术，而是让卡牌、牌堆、弹窗、角色站位和背景占位在同一套尺寸语言里迭代。

## 基准

- 设计基准分辨率：`1280x720`
- 基础网格：`8px`
- 布局网格：`16px`
- 主要边距：`10px` 到 `16px`
- UI 资源优先按 `8px` 整数倍制作和摆放。

## 战斗屏幕分区

```text
1280x720

top hud:        y 0    - 72
enemy stage:    y 78   - 260
combat center:  y 260  - 360
dice lane:      y 360  - 460
hand lane:      y 480  - 710
```

## 底部区域

底部不再让手牌铺满全宽，而是拆成三个槽位：

```text
left pile slot:   x 10   - 96
hand slot:        x 104  - 1092
right pile slots: x 1100 - 1270
```

- 抽牌堆：`86x110`
- 弃牌堆：`86x110`
- 消耗堆：`86x110`
- 手牌显示卡片：`150x225`
- 手牌卡片之间建议间距：`6px` 到 `8px`

## 卡牌尺寸

当前 `CardView.tscn` 使用高精度 UI 母版：

- 卡牌母版：`512x768`
- 宽高比：`2:3`
- 手牌显示缩放：`0.78125`
- 手牌显示尺寸：`150x225`
- 弹窗显示缩放：`0.74`
- 弹窗卡片格子：`148x222`

后续 AI 主图建议按卡牌母版的主图槽位准备：

- 主图槽位：约 `448x320`
- 若先做像素占位：可用 `224x160` 后按 `Nearest` 放大 2 倍。
- 若用 AI 先出概念图：建议输出更高分辨率，再手动裁切到同一比例。

## 弹窗

牌包、抽牌堆、弃牌堆、消耗堆统一使用同一类浏览规格：

- 弹窗尺寸：`1040x600`
- 卡片区域：左侧滚动网格
- 文字预览：右侧 `300px`
- 网格列数：`4`
- 弹窗卡片：`148x222`
- 横向滚动：禁用，只允许纵向滚动

单击卡片只更新右侧文字预览；双击是否移动卡片由调用方决定。

## 训练场背景占位

训练场背景建议先按低分辨率像素资源规划：

- 背景母图：`320x180`
- 显示放大：`4x`
- 对应屏幕：`1280x720`
- 像素颗粒：`4x4` 屏幕像素为一个原始像素。

第一版背景只需要：

- 深色地面/墙面
- 角色站位平台
- 敌人站位平台
- 中央能量或骰子测试区
- 轻量网格线

正式战斗闭环稳定前，美术资源只作为占位，不应阻塞战斗逻辑验证。
